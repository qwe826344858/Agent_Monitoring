# Agent Teams 模块化交付技能

## 执行原则

1. 先完整理解目标文档与边界，不跳过需求澄清。
2. 始终由 Leader 统一拆分模块、定义依赖、分配负责人。
3. 开发与测试并行推进，但以模块验收为合并门禁。
4. 每个模块必须“代码 + 注释 + 测试用例 + 结果记录”四项齐全。
5. 无法解决的问题必须落文档，不允许口头遗留。

## 标准流程

1. 理解阶段：读取目标文档，抽取功能模块、约束、验收标准。
2. 拆分阶段：Leader 输出模块任务表（模块ID、目标、输入输出、依赖、负责人、截止时间）。
3. 开发阶段：开发Agent按模块实现，遵循编码规范与注释规范，补齐测试用例。
4. 自测阶段：开发Agent本地执行测试并记录结果。
5. 验证阶段：测试Agent复测模块，更新测试报告并给出通过/不通过结论。
6. 归档阶段：更新开发进度文档（完成度、风险、阻塞项、下一步）。
7. 异常阶段：若卡住，新增《无法解决的问题清单.md》到目标文档目录。

## 交付物要求

1. `模块任务分配表.md`
2. `开发进度文档.md`（每次模块完成后立即更新）
3. `测试报告.md`（测试Agent维护）
4. `无法解决的问题清单.md`（仅在出现阻塞且无法闭环时创建）

## 验收门禁

1. 模块代码符合规范并含必要注释。
2. 模块测试用例齐全且全部通过。
3. 测试Agent复核通过并在测试报告留痕。
4. 开发进度文档与实际状态一致。
5. 未解决问题必须可追踪（问题、影响、临时方案、责任人、计划时间）。
