# 模块任务分配表

## 项目：Agent Teams 实时监控面板

| 模块ID | 模块名称 | 目标 | 输入 | 输出 | 依赖 | 负责人 |
|--------|---------|------|------|------|------|--------|
| M1 | 数据模型与基础设施 | 定义所有数据结构 + Go 项目初始化 + 依赖引入 | 设计方案 §四 数据源定义 | `internal/model/types.go`, `go.mod`, `go.sum` | 无 | backend-core |
| M2 | 文件监听与数据读取 | 实现 fsnotify 文件监听 + JSONL 解析 + 变化检测 | M1 的类型定义 | `internal/watcher/watcher.go`, `internal/token/tracker.go` | M1 | backend-core |
| M3 | HTTP API + WebSocket | Gin 路由 + REST API + WebSocket 推送 + 主入口 | M1 类型 + M2 watcher | `internal/handler/api.go`, `internal/handler/ws.go`, `main.go` | M1, M2 | backend-api |
| M4 | 前端页面 | 四面板 Dashboard（成员卡片、任务看板、Token 统计、通信流） | 设计方案 §六 前端设计 | `static/index.html`, `static/style.css`, `static/app.js` | M3（API 接口定义） | frontend-dev |
| M5 | 集成测试与验证 | 端到端验证：启动服务 → 创建模拟数据 → 验证页面展示 | 所有模块 | 测试报告 | M1-M4 | tester |

## 执行顺序

```
M1（数据模型）
  ↓
M2（文件监听）──→ M4（前端，可与 M3 并行，先按接口约定开发）
  ↓
M3（API + WebSocket）
  ↓
M5（集成测试）
```

## 验收标准

1. `go build` 编译通过，生成单二进制
2. 启动后 `http://localhost:8080` 可访问 Dashboard
3. 在 `~/.claude/teams/` 下创建模拟团队数据，页面实时展示成员、任务、消息
4. WebSocket 推送延迟 < 500ms
5. Token 统计面板每 5 秒刷新
